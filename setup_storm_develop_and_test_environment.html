Storm测试环境安装

依赖

  • python2.7
  • oracle jdk6 (并已经设置好了JAVA_HOME变量)
  • libtool
  • make
  • gcc
  • libuuid-dev

环境变量假设

假设当前操作用户为user,则假定如下变量：

LOCAL_DIR=$HOME/.local
OP_DIR=$HOME/software/storm-environment

Zookeeper

安装代码:

cd $OP_DIR
wget "http://mirrors.hust.edu.cn/apache/zookeeper/zookeeper-3.4.5/zookeeper-3.4.5.tar.gz"
tar -zxvf zookeeper-3.4.5.tar.gz
mv zookeeper-3.4.5 $LOCAL_DIR

打开$HOME/.bashrc文件设置ZOOKEEPER_HOME和修改PATH环境变量：

export ZOOKEEPER_HOME=$HOME/.local/zookeeper-3.4.5
export PATH=$ZOOKEEPER_HOME/bin:$PATH

使环境变量生效：

source $HOME/.bashrc

单实例配置：

cd $ZOOKEEPER_HOME/conf
cp zoo_sample.cfg zoo.cfg

启动实例：

zkServer.sh

Storm

zeromq 2.1.7

按storm文档的建议，需要编译安装2.1.7版本的zeromq，其它版本可能有问题。此版本较老，为了不影响系统zeromq环境，单独编译安装：

cd $OP_DIR
wget http://download.zeromq.org/zeromq-2.1.7.tar.gz
tar -xzf zeromq-2.1.7.tar.gz
cd zeromq-2.1.7
./configure --prefix=$LOCAL_DIR
make
make install

jzmq

cd $OP_DIR
git clone https://github.com/nathanmarz/jzmq.git
cd jzmq
./autogen.sh
./configure --prefix=$LOCAL_DIR --with-zeromq=$LOCAL_DIR
make
touch src/classdist_noinst.stamp
make
javac cd src/org/zeromq/*.java
make
make install

storm

cd $OP_DIR
wget "https://www.dropbox.com/s/fl4kr7w0oc8ihdw/storm-0.8.2.zip"
unzip storm-0.8.2.zip
mv storm-0.8.2 $LOCAL_DIR

打开$HOME/.bashrc,设置环境变量：

export STORM_HOME=$HOME/.local/storm-0.8.2
export PATH=$STORM_HOME/bin:$PATH

使环境变量生效：

source $HOME/.bashrc

Storm开发环境安装

eclipse

安装eclipse的方法参考网上的教程，待eclipse安装好后，再安装内置maven插件。注意，此插件只用来导入maven项目，真正的编译还是用命令行工具（内置的maven插件折腾了一阵，一直有问题）

maven

安装maven

sudo apt-get install maven

配置maven代理 编辑$HOME/.m2/settings.xml，加入如下内容：

<proxies>
    <proxy>     
        <active>true</active> 
        <protocol>http</protocol> 
        <host>127.0.0.1</host> 
        <port>8087</port> 
        <nonproxyhosts>*github.com*|*clojars.org*|*1verge.net*</nonproxyhosts> 
    </proxy>
</proxies>

根据自己的情况修改上述配置中的代理地址及不需要代理的host。如果自己的网络条件不需要代理，就可以访问源，那么可以将active一项改为false。根据本人使用情况，有一些源不用代理速度奇慢，甚至无法访问。
